## 1. 产品背景与目标

### 1.1 背景

大量中小企业在品牌建设初期需要一套专业的 VI（视觉识别）手册，但委托专业设计公司成本高、周期长，且沟通成本大。同时市场上缺乏面向供应商侧的 AI 辅助 VI 生成工具。

### 1.2 产品定位

**B2B 代运营服务产品**。客户通过极简界面提交品牌素材与需求，我方运营团队借助 AI 辅助完成 VI 手册的设计与交付。产品的核心用户并非终端企业，而是我方内部的设计与运营团队。

### 1.3 商业目标

- 降低 VI 手册设计的单客交付成本，缩短交付周期
- 通过 AI 辅助实现规模化接单
- 积累品牌资产库，形成行业壁垒

## 2. 目标用户与角色

### 2.1 角色定义

| 角色 | 说明 | 使用端 |
|---|---|---|
| **客户 (Client)** | 有 VI 设计需求的企业代表，提交素材与需求，接收最终手册 | 客户端 |
| **客服/销售 (Sales)** | 处理咨询留资，跟进客户，确认需求 | 管理端 |
| **设计师 (Designer)** | 使用 AI 辅助工具完成 VI 手册的设计、精修与交付 | 管理端 |
| **管理员 (Admin)** | 管理项目分配、查看运营数据、维护内部素材库 | 管理端 |

### 2.2 用户故事

| 编号 | 作为... | 想要... | 以便... |
|---|---|---|---|
| US-01 | 客户 | 快速提交品牌 Logo 和 IP 公仔素材 | 启动 VI 设计流程 |
| US-02 | 客户 | 上传参考 VI 手册供 AI 学习风格 | 确保设计方向符合预期 |
| US-03 | 客户 | 随时查看项目进度 | 了解当前处于哪个阶段 |
| US-04 | 销售人员 | 查看新提交的咨询留资 | 及时跟进潜在客户 |
| US-05 | 设计师 | 让 AI 基于客户素材快速生成多套 VI 方案 | 提高初稿产出效率 |
| US-06 | 设计师 | 让 AI 参考客户上传的 VI 手册生成方案 | 快速对齐客户偏好风格 |
| US-07 | 设计师 | 在编辑器中微调 AI 方案 | 确保最终交付质量 |
| US-08 | 设计师 | 收藏好的 AI 方案 | 后续复用或给相似行业参考 |
| US-09 | 管理员 | 查看所有项目的进度看板 | 掌握团队产能和交付情况 |

```
┌──────────────────────────────────────────────────────────────┐
│                        客户端 (Client Portal)                  │

│  进度查询 ←━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━│
└──────────────────────────┬───────────────────────────────────┘
                           │ HTTPS API
┌──────────────────────────▼───────────────────────────────────┐
│                      API 网关 (Next.js API Routes)              │
└──────────┬──────────┬──────────┬─────────┬───────────────────┘
           │          │          │         │
    ┌──────▼──┐ ┌────▼───┐ ┌───▼───┐ ┌──▼────┐
    │ PostgreSQL│ │ S3/R2 │ │ Redis │ │ AI    │
    │          │ │ 存储   │ │ 队列   │ │ API   │
    └──────────┘ └────────┘ └───────┘ └───────┘

┌──────────────────────────────────────────────────────────────┐
│                    管理端 (Admin Console)                      │
│  工作台 → 项目管理 → 素材管理 → AI方案生成 → VI编辑器        │
```

### 3.2 核心流程

```
客户提交需求 ──→ 销售跟进确认 ──→ 设计师使用AI生成方案
                            ↓
                    设计师编辑精修 ──→ 内部审核
                            ↓
                    导出交付 ──→ 通知客户
```

## 5. 客户端需求详情

### 5.1 首页 (Homepage)

#### 5.1.1 页面目标
向潜在客户传递服务价值、建立信任、引导完成需求提交。

#### 5.1.2 内容模块

| 模块 | 内容 | 说明 |
|---|---|---|
| **Hero 区域** | 主标题 + 副标题 + CTA 按钮 | 主标题示例：*"上传您的 Logo，AI 为您生成专业 VI 手册"* |
| **服务流程** | 3-4 步流程示意（图文） | 提交素材 → AI 分析 → 专业设计 → 交付手册 |
| **案例展示** | 已交付客户案例轮播 | 展示前后对比或成品手册截图，配备客户评价 |
| **服务优势** | 3-4 个核心优势卡片 | 如：AI 加速、专业团队、行业经验、保密保障 |
| **常见问题** | FAQ 折叠面板 | 价格、周期、流程、版权等常见疑问 |
| **底部** | 联系方式 + CTA | 引导提交需求 |

#### 5.1.3 页面状态

| 状态 | 处理 |
|---|---|
| 加载态 | Skeleton 骨架屏，按模块依次淡入 |
| 空态 | 无（首页为静态页） |
| 错误态 | 图片加载失败显示占位图 |
| 边界情况 | 移动端适配：卡片式布局，流程步骤改为纵向 |

---

### 5.2 咨询留资 (Consultation)

#### 5.2.1 功能描述
客户填写需求信息并上传品牌素材的表单页面，是客户端的核心功能入口。

#### 5.2.2 表单字段

| 字段 | 类型 | 必填 | 校验规则 |
|---|---|---|---|
| 联系人姓名 | 文本输入 | 是 | 2-20 字符 |
| 公司名称 | 文本输入 | 是 | 2-100 字符 |
| 联系电话 | 文本输入 | 是 | 11 位手机号或固定电话 |
| 微信号 | 文本输入 | 否 | - |
| 所属行业 | 下拉选择 | 是 | 选项：科技/互联网、餐饮/食品、零售/电商、教育/培训、医疗/健康、金融/法律、文化/传媒、制造业、其他 |
| 预算范围 | 下拉选择 | 否 | 选项：3000以下、3000-5000、5000-10000、10000-30000、30000以上、待定 |
| 需求描述 | 多行文本 | 否 | 最长 500 字 |

#### 5.2.3 文件上传区

**LOGO 上传**

| 项目 | 说明 |
|---|---|
| 上传方式 | 点击或拖拽 |
| 格式限制 | PNG / JPG / SVG / AI / EPS |
| 文件大小 | 单文件 ≤ 20MB |
| 数量 | 最多 5 个 |
| 展示 | 缩略图网格，可删除 |
| 边界情况 | 格式不符时 toast 提示；大小超限时提示；拖拽区无文件时显示引导文案 |

**IP 公仔 / 吉祥物上传**

| 项目 | 说明 |
|---|---|
| 上传方式 | 点击或拖拽 |
| 格式限制 | PNG / JPG / GIF |
| 文件大小 | 单文件 ≤ 20MB |
| 数量 | 最多 10 个 |
| 附带信息 | 填写 IP 名称（必填，最多 20 字符）、性格描述（选填，最多 100 字符） |
| 展示 | 缩略图 + 名称标签网格 |
| 边界情况 | 同 LOGO 上传校验 |

**参考 VI 手册上传（新增核心功能）**

| 项目 | 说明 |
|---|---|
| 上传方式 | 点击上传 |
| 格式限制 | PDF |
| 文件大小 | ≤ 50MB |
| 数量 | 最多 1 个 |
| 参考选择 | 勾选框：*「AI 生成时参考本手册的风格」*（默认勾选） |
| 展示 | 文件名 + 页数 + 上传时间；可删除替换 |
| 边界情况 | 非 PDF 格式提示；超过 50MB 提示；PDF 页数过多（>100页）提示可能耗时较长 |

#### 5.2.4 表单状态

| 状态 | 处理 |
|---|---|
| 加载态 | 提交按钮 loading + 禁用；文件上传显示进度条 |
| 成功态 | 跳转确认页，显示项目编号和后续流程说明 |
| 错误态 | 字段高亮显示错误信息；文件上传失败显示重试按钮 |
| 空态 | 表单初始化全空，上传区显示"点击或拖拽文件到此处" |
| 边界情况 | 未选择行业时提交 → 提示必填；超大文件上传 → 提前校验阻止；长时间上传 → 超时处理 |

#### 5.2.5 确认页

| 元素 | 说明 |
|---|---|
| 成功标识 | 对勾图标 + "提交成功" 文案 |
| 项目编号 | 唯一编号，格式 VI-20260523-XXXX |
| 提交摘要 | 汇总展示所有已提交的信息和素材缩略图 |
| 后续说明 | 文案："我们将在 1-2 个工作日内联系您确认需求" |
| CTA | "查看项目进度"按钮（跳转进度查询页） |

| 项目 | 说明 |
|---|---|
| 入口 | 首页底部 / 确认页按钮 |

### 6.3 AI 方案生成

#### 6.3.1 功能描述
设计师在项目中点击「AI 生成方案」，AI 基于客户素材生成多套 VI 方案供选择。

#### 6.3.2 生成参数

| 参数 | 说明 | 默认值 |
|---|---|---|
| 参考模式 | 强参考 / 弱参考 / 不参考 | 弱参考 |
| 参考源 | 客户上传手册 / 内部素材库 | 客户上传手册 |
| 生成数量 | 3 / 5 套 | 3 套 |
| 风格倾向 | 极简 / 科技 / 传统 / 潮酷 / 自然 | 根据行业自动推荐 |
| 色彩策略 | 提取 Logo 色 / 行业推荐色 / 参考手册色 | 提取 Logo 色 |

#### 6.3.3 页面状态

| 状态 | 处理 |
|---|---|
| 加载态 | 生成中显示进度动画 + "AI 正在分析素材..." |
| 成功态 | 展示生成的方案卡片网格，每张卡片显示缩略图 + 风格标签 |
| 错误态 | AI 生成失败 → "生成失败，请重试" + 错误原因 |
| 空态 | 首次进入，显示"点击开始 AI 生成" 引导 |
| 边界情况 | 参考手册 PDF 解析失败 → 提示并降级为不参考模式；生成时间超过 30s → 后台队列异步 |

#### 6.3.4 操作

| 操作 | 说明 |
|---|---|
| 选择方案 | 点击卡片设为主方案 |
| 收藏方案 | 星标收藏，供后续复用 |
| 重新生成 | 调整参数后再次生成 |
| 对比 | 并排对比 2-3 套方案 |

### 6.4 收藏

#### 6.4.1 使用场景
收藏功能主要面向**管理端的设计师**。在 AI 生成的多套方案中，设计师可以收藏满意的方案，供后续复用或作为内部参考模板。

#### 6.4.2 功能

| 功能 | 说明 |
|---|---|
| 收藏 / 取消收藏 | AI 方案卡片上的星标按钮，点击切换 |
| 收藏列表 | 独立页面展示所有收藏方案 |
| 筛选 | 按行业 / 风格 / 收藏日期筛选 |
| 搜索 | 按方案名称搜索 |
| 查看详情 | 点击收藏项跳转至原项目或方案详情 |
| 批量管理 | 批量取消收藏、批量导出 |

#### 6.4.3 数据存储

```
收藏记录 = {
  id: string,
  planId: string,
  projectId: string,
  clientIndustry: string,
  styleLabel: string,
  thumbnailUrl: string,
  collectedBy: string,    // 设计师 ID
  collectedAt: timestamp,
  notes: string           // 设计师备注（可选）
}
```

#### 6.4.4 页面状态

| 状态 | 处理 |
|---|---|
| 加载态 | 卡片 Skeleton |
| 空态 | "还没有收藏任何方案，在 AI 生成方案时点击星标收藏" |
| 错误态 | 加载失败 → 重试 |
| 边界情况 | 收藏数超过 100 → 分页；搜索无结果 → "未找到匹配的收藏" |

### 6.6 AI 对话助手
