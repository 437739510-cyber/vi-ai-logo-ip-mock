# 任务拆分计划 (Task Splitting)

> 对应教程第 05 步：将项目拆成独立任务窗口，每次只做一个，做完验证后再进下一个。

---

## 顺序策略

**一个做完再做下一个**，不会同时跑多个任务。优先级按依赖关系排列：

```
窗口 A (客户端) ──→ 窗口 B (管理端框架) ──→ 窗口 C (AI 生成+收藏) ──→ 窗口 D (编辑器+导出)
```

每个窗口完成后都会提醒你提交 Git 存档，再进入下一个。

---

## 窗口 A：客户端页面

**目标**：实现客户能看到的所有页面，这是客户的第一印象，也是项目的前提。

| 子任务 | 说明 | 预计文件数 |
|---|---|---|
| A-1 | 项目脚手架初始化（Next.js 15 + 依赖安装 + 目录结构） | ~10 |
| A-2 | 共享基础设施（TypeScript 类型 + Mock 数据 + 通用组件 + 布局） | ~15 |
| A-3 | 首页（Hero + 服务流程 + 案例 + FAQ + CTA） | ~6 |
| A-4 | 咨询留资页（完整表单 + Logo/IP/参考手册上传 + 校验） | ~4 |
| A-5 | 提交确认页 + 进度查询页 | ~3 |

**验证方式**：
```bash
npm run build && npm run dev
# 访问 http://localhost:3000 查看首页
# 访问 http://localhost:3000/consultation 查看表单
```

**不能碰的范围**：管理端所有页面、AI 生成逻辑、编辑器

---

## 窗口 B：管理端框架

**目标**：搭建我方使用的后台框架，能看项目、管项目。

| 子任务 | 说明 | 预计文件数 |
|---|---|---|
| B-1 | 工作台（统计卡片 + 最近动态） | ~3 |
| B-2 | 项目列表（筛选 + 搜索 + 分页） | ~4 |
| B-3 | 项目详情页（查看客户提交的全部素材） | ~4 |

**验证方式**：
```bash
npm run build && npm run dev
# 访问 http://localhost:3000/admin/dashboard
# 访问 http://localhost:3000/admin/projects
```

**不能碰的范围**：AI 生成、编辑器、导出功能

## 窗口 C：AI 方案生成 + 收藏

**目标**：实现 AI 根据客户素材生成 VI 方案（含参考模式选择），以及收藏功能。

| 子任务 | 说明 | 预计文件数 |
|---|---|---|
| C-1 | AI 方案生成页（参数配置 + 生成按钮 + 进度动画 + 结果卡片） | ~5 |
| C-2 | 参考模式选择器（强/弱/不参考） | ~1 |
| C-3 | 收藏功能（星标 + 收藏列表 + 筛选 + 批量管理） | ~4 |

**验证方式**：
```bash
npm run build && npm run dev
# 访问 http://localhost:3000/admin/projects/xxx/generate
# 访问 http://localhost:3000/admin/favorites
```

**不能碰的范围**：VI 编辑器、导出、客户管理

**验证方式**：

# 访问 http://localhost:3000/admin/preview/xxx

**不能碰的范围**：（此窗口为最后一个，无范围限制）

## 任务切换说明

每次切换窗口时，我会做两件事：

1. **提醒你 Git 存档** — 你复制粘贴我给的命令就行
2. **总结当前进度** — 一句话说清刚才做完了什么、下一步做什么
