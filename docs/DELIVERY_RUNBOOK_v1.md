## 1. 客户接入流程

### 1.1 客户来源

| 来源 | 说明 | 负责人 |
|------|------|--------|
| 创业基地推荐 | 松山湖海峡两岸青年创业基地 | 商务 |
| 主动咨询 | 网站/社交媒体 | 商务 |
| 合作伙伴引荐 | 设计公司/品牌顾问 | 商务 |
| 椰岛工坊同类 | 食品饮料行业客户 | 商务 |

### 1.2 初次沟通清单

```
□ 了解客户行业和品牌现状
□ 说明 Brand Brain 能做什么（展示案例）
□ 确认客户需求匹配度
□ 确定试点模式（Professional / Assisted / Discovery）
□ 确认价格意向
□ 预约信息收集时间
```

### 1.3 客户筛选标准

```
必要条件：
  □ 有明确品牌建设需求
  □ 有基本的品牌描述或想法
  □ 愿意提供反馈

加分条件：
  □ 有 Logo / IP 资产
  □ 属于已验证行业（food_beverage 优先）
  □ 愿意推荐给同行
```

## 2. 信息收集流程

### 2.1 客户信息表

| 字段 | 示例 | 说明 |
|------|------|------|
| 公司名称 | 椰岛工坊 | 必填 |
| 所属行业 | food_beverage | 必填 |
| 品牌描述 | 北纬18度黄金产区的椰子水定制专家 | 必填，越详细越好 |
| 产品类别 | coconut_water | 选填 |
| 目标受众 | 高端酒店采购、健身博主 | 选填 |
| 品牌个性 | 自然、健康、匠心、热带、阳光、专业 | 选填 |
| 已有 Logo | ✓ | 选填，有则上传 |
| 已有 IP 公仔 | ✓ | 选填，有则上传 |
| 参考案例 | 竞品/灵感链接 | 选填 |

### 2.2 信息收集方式

```
方式 A：直接填写（Professional 模式）
  客户直接提供上述信息
  预计耗时：10分钟

方式 B：引导式填写（Assisted 模式）
  通过问答引导客户梳理品牌信息
  预计耗时：20-30分钟

方式 C：采访式（Discovery 模式）
  AI 主动提问，客户回答
  预计耗时：15-20分钟
```

### 2.3 信息质量要求

```
最低要求：
  □ 公司名称
  □ 行业分类
  □ 品牌描述（至少一句话）

推荐要求：
  □ 目标受众
  □ 品牌个性（至少3个词）
```

## 3. Full Mode 执行流程

### 3.1 前置检查

```
执行前确认：

□ .env.local 配置完整
  - NEXT_PUBLIC_SUPABASE_URL
  - NEXT_PUBLIC_SUPABASE_ANON_KEY
  - SUPABASE_SERVICE_KEY
  - DEEPSEEK_API_KEY
  - ALIYUN_API_KEY

□ NEXT_PUBLIC_MEMORY_ADAPTER=supabase
□ Supabase 4 表已创建
□ Supabase Storage bucket 正常
□ Build 通过（npm run build）
```

### 3.2 执行命令

```bash
# 设置生产模式
export NEXT_PUBLIC_MEMORY_ADAPTER=supabase

# 运行 Full Mode
npx tsx --env-file=.env.local src/lib/memory/__tests__/yedao-full-mode-validation.ts
```

### 3.3 预计耗时

| 阶段 | 预计耗时 | 说明 |
|------|---------|------|
| 信息处理 | 10-30分钟 | 取决于信息完整度 |
| Pipeline 执行 | 2-3秒 | AI API 调用 |
| Manual Composer | ~40ms | 页面生成 |
| 结果整理 | 5-10分钟 | 输出报告 |
| **总耗时** | **~30分钟** | |

### 3.4 执行监控要点

```
执行中关注：
  □ 每个 Agent 是否成功（共 6 个）
  □ Manual Composer 页数（期望：15-25 页）
  □ Quality Score（期望：≥ 70）
  □ 是否出现 401/500/Timeout
  □ Memory 是否写入成功
```

## 4. 交付流程

### 4.1 交付物清单

```
□ Brand Analysis Report（品牌分析报告）
  - 品牌类型 / 置信度 / 视觉方向
  - IP 策略建议

□ Module Plan（模块规划）
  - 15 模块清单（含优先级）
  - 套餐推荐

□ Design Direction（视觉方向）
  - 风格关键词
  - 色彩策略（主色/辅助色/强调色）
  - 字体建议

□ Generated Manual Pages（生成手册页）
  - 23页手册（可在线预览）

□ Quality Score Report（质量评分）
  - 总分 87/100
  - 各维度评分
```

### 4.2 交付方式

```
□ 在线预览链接（Vercel 部署后）
□ PDF 导出（如有需要）
□ 案例报告（CASE_STUDY 格式）
```

### 4.3 交付后跟进

```
交付后 24 小时内：
  □ 发送客户反馈表（PILOT_FEEDBACK_TEMPLATE）
  □ 确认客户是否理解输出内容
  □ 记录客户评价

交付后 1 周内：
  □ 跟进客户使用情况
  □ 收集改进建议
  □ 询问推荐意向
```

## 5. 售后反馈流程

### 5.1 反馈收集

```
使用 PILOT_FEEDBACK_TEMPLATE_v1 记录：
  □ 整体满意度（1-5）
  □ 品牌分析准确性
  □ 输出质量
  □ 产品体验
  □ 付费意愿
  □ 推荐意愿
  □ 改进建议
```

### 5.2 问题处理

```
严重问题（阻断交付）：
  → 当天修复
  → 重新运行 Pipeline

一般问题（体验优化）：
  → 记录到 v1.1 规划
  → 下次迭代处理

建议/需求：
  → 评估可行性
  → 纳入产品路线
```

### 5.3 客户分级

```
A 级：非常满意 + 愿意付费 + 愿意推荐
  → 重点跟进 → 转化为付费客户

B 级：满意 + 有条件付费
  → 持续跟进 → 提供额外价值

C 级：一般满意
  → 收集反馈 → 改善产品
```
