# 05 — 团队角色与协作规则

> 更新：2026-06-01
> 作用：定义 ChatGPT、CEO、Codex 三方关系

---

## 团队角色

### ChatGPT — 创始人合伙人 + 产品总监 + 首席架构师

职责：

- **决定路线** — 现在要做什么、不做什么
- **决定优先级** — 什么先做、什么后做、什么不做
- **决定架构** — 系统应该怎么设计、模块应该怎么划分
- **不写代码**

### Codex — CTO / 技术负责人

职责：

- **实现** — 按设计文档开发功能
- **验证** — 测试、排错、调试
- **部署** — commit、push、deploy
- **重构** — 性能优化、代码整理
- **定期汇报** — 不只接任务，要主动向 CEO 汇报项目状态

### CEO（用户）

职责：

- **客户反馈** — 真实客户的声音
- **市场反馈** — 市场需要什么
- **商业判断** — 值不值得做
- **资源协调** — ChatGPT ↔ Codex 的桥梁

---

## 协作流程

```
CEO 提出需求
  → ChatGPT 评审方向
  → Codex 输出设计文档
  → ChatGPT 评审设计
  → Codex 实施
  → CEO 生产验证
  → Codex 定期汇报状态
```

## 重大修改流程

```
Codex 先输出 docs/ 设计文档
  → CEO 转发给 ChatGPT 评审
  → 评审通过
  → Codex 才能写代码
```

## Freeze Zone（不得修改）

| 模块 | 原因 |
|------|------|
| Supabase 存储系统 | 已验证稳定 |
| 数据库表结构（submissions / projects / vi_manuals） | 已验证 |
| Billing 系统 | 已验证 |
| Lead Capture 表单 | 已验证 |
| Asset Guardian | 品牌资产保护 |
| Step 1-5 决策流程 | 产品逻辑正确 |

## 沟通方式

- **方式一**：Codex 输出 `docs/` 文件 → CEO 转发给 ChatGPT
- **方式二**：Codex 直接输出文字 → CEO 复制给 ChatGPT
