## 品牌分析框架

Brand Brain 的品牌分析应该覆盖以下维度：

| 维度 | 问题 | 对应 Brand Brain 组件 |
|------|------|----------------------|
| 品牌类型 | 消费品牌/科技品牌/服务品牌？ | BrandType |
| 行业归属 | 餐饮/科技/零售？ | IndustryCategory |
| 品牌人格 | 自然/创新/精致/潮流？ | BrandPersona |
| 视觉方向 | 自然有机/极简现代/大胆活力？ | VisualDirection |
| 目标受众 | 给谁看？ | TargetAudience |
| 品牌定位 | 一句话说清你是谁 | brandPositioning |
| 已有资产 | 有 Logo 吗？有 IP 吗？ | hasLogo / hasMascot |

**原则：** 以上信息必须在进入套餐推荐前完成。不跳过分析直接推荐套餐。

## 品牌成长路径

对应 Brand Brain 的 Package System：

| 阶段 | 适用场景 | 推荐套餐 | 页数 |
|------|---------|---------|------|
| 初创 | 刚成立，需要基础规范 | basic_vi | 12-18 |
| 成长 | 有基础，需要统一升级 | brand_vi | 18-30 |
| 连锁/消费 | 多门店，需要完整体系 | brand_ip | 30-50 |

**但不只看品牌特征。** Package Recommendation 是双因子评分：

```
finalScore = brandScore(profile) × 0.6 + businessScore(business) × 0.4
```

即：品牌适合什么 × 客户现在需要什么。

## IP 策略判断逻辑

| 条件 | 输出 | 行为 |
|------|------|------|
| 已有 IP | protect_existing | 保护、规范、延展，不重绘 |
| 无 IP + 消费/连锁/餐饮 | create_new | 创建品牌 IP |
| 无 IP + 科技/专业服务 | optional_recommend | 根据品牌人格判断 |
| 无 IP + 法律/金融/制造 | not_needed | 不建议创建 |

## 品牌手册的结构

一个专业的企业级 VI 手册应包含：

### 基础部分（Basic VI）

- Logo 规范（标识、比例网格、最小尺寸、保护空间）
- 颜色体系（标准色、辅助色、色彩优先级）
- 字体体系（专用字体、通用字体）
- 错误用法（禁止的 Logo 使用方式）
- 背景控制（浅色/深色/图片背景）

### 应用部分（Application）

- 办公系统（名片、信封、PPT 模板）
- 资料封面
- 社媒头像
- 宣传物料
- 包装系统
- 门店/空间应用

### IP 部分（IP Manual）

- 角色设定
- 三视图
- 标准色
- 表情/动作系统
- 场景延展
- 物料应用
