# 05 — 反面案例库（ANTI_PATTERN_LIBRARY）

> 来源：两份"小智猫"反面案例 PDF
> 更新：2026-05-31
> 警告：以下案例指出了 AI 生成 IP 手册的典型问题。这些是"不应该做什么"的参考。

---

## 假 IP 的五大特征

### 1. 看起来像手册，但缺少真实规范

反面案例的问题：页面排版精美，有封面、目录、章节页，像一本专业手册。但翻到具体页面，内容只是视觉展示，没有规范说明。

**判断标准：**
- 有页面，但没有尺寸说明
- 有展示，但没有色值标注
- 有图，但没有比例/网格/保护空间

### 2. IP 介绍很多，但没有生产标准

- 有 IP 角色的名称和描述
- 但没有三视图比例
- 没有标准色值和配色比例
- 没有造型基因拆解
- 没有最小尺寸限制

### 3. 有视觉展示，但缺少资产管理能力

反面案例中 IP 的各个角度看起来风格一致，但实际上没有统一的规范文档指导后续生产。

**判断标准：** 如果让另一个设计师照着这本手册制作物料，能不能做出一致的结果？不能，就是假规范。

### 4. 可能重新设计了原始 IP

在 AI 生成过程中，IP 的以下特征可能被改变：
- 面部比例
- 颜色倾向
- 核心元素的位置/形状
- 整体风格（从扁平变立体，或从 3D 变 2D）

**这是 Asset Guardian 必须存在的原因。**

### 5. 很多页面是概念介绍，不是 VI 规范

反面案例中很大比例是"IP 故事"、"角色设定"、"场景概念"，而不是"使用规范"、"禁止事项"、"应用规则"。

**好手册的比例：**

```
30% 概念/介绍/故事
70% 规范/规则/标准/应用
```

反面案例的比例几乎是反过来的。

## 品牌人格 ≠ IP 人格

这是一个高频错误。

| 维度 | 品牌人格（智方增长科技） | IP 人格（小智猫） |
|------|------------------------|------------------|
| 角色 | 企业品牌 | 品牌人格化入口 |
| 风格 | 专业、科技、前沿 | 聪明、可靠、温暖 |
| 色彩 | 科技蓝 | 科技蓝 + 像素眼 + 黑屏 |
| 表达 | 正式、权威 | 亲和、陪伴、可对话 |

**正确关系：** IP 人格是品牌人格的**具象化 + 情感化**，不是替代或重新定义。

## Character Integrity Check（角色完整性检查）

每一页 AI 生成的 IP 内容都应该通过以下检查：

### 视觉一致性检查

- [ ] IP 颜色是否与品牌标准色一致？
- [ ] IP 核心元素（猫耳、面屏、像素眼）是否全部存在？
- [ ] IP 比例是否与三视图保持一致？
- [ ] 是否有被拉伸或变形？
- [ ] 是否有颜色偏移？

### 规范完整性检查

- [ ] 是否有尺寸/比例标注？
- [ ] 是否有色值标注（HEX/RGB/CMYK）？
- [ ] 是否有最小使用尺寸？
- [ ] 是否有保护空间标注？
- [ ] 是否有禁止用法？

### 资产管理检查

- [ ] 原始 IP 是否被重绘？
- [ ] 是否使用了受保护资产的源文件？
- [ ] 是否在背景 prompt 中禁止了重绘？

## 常见 AI 生成错误

### 1. 风格漂移

同一 IP 在不同页面风格不一致。例如：第一页扁平风格，第三页变成伪 3D。

**解决：** 通过 seedAssetId 保持一致性；使用明确的 style 参数。

### 2. 文字幻觉

AI 在图片中生成无意义的文字或乱码。

**解决：** 所有文字应在 SVG 层添加，不在图片生成 prompt 中要求文字。

### 3. 元素丢失

多张图生成时，IP 的某个核心元素（如铃铛、耳朵）可能随机消失。

**解决：** 每张图独立验证 Character Integrity Check。

### 4. 背景污染

AI 在背景中生成不应存在的元素（例如在背景中画了一个假的 Logo 变体）。

**解决：** 在负面 prompt 中明确禁止；使用原图 `<image>` 引用。

### 5. 规范页变成介绍页

"Logo 规范"页面变成了"Logo 展示"页面。

**判断标准：** 这一页是否告诉读者"可以怎么做"和"不可以怎么做"？如果只是展示，就不是规范。

## Quality Score 与反面案例的对应关系

| Quality Score 维度 | 反面案例反映的问题 | 扣分标准 |
|-------------------|-------------------|---------|
| brandLogic | 品牌人格与 IP 人格不一致 | < 15 分标记风险 |
| visualConsistency | 风格漂移、颜色偏移 | < 15 分标记风险 |
| assetProtection | IP 被重绘、核心元素丢失 | < 15 分为 critical |
| guidelineCompleteness | 有页面没有规范 | < 15 分为 fake_guideline_risk |
| productionReadiness | 缺少尺寸/色值/比例 | < 10 分标记风险 |

## 反面案例总结

两个小智猫反面案例的共同问题：

1. **有 IP 手册的形式，没有 IP 手册的内容** — 页面看起来专业，但缺少真正的规范
2. **IP 被重新设计过** — 核心元素发生风格变化，不是原始 IP 的规范化延展
3. **缺少生产标准** — 没有三视图比例、标准色值、最小尺寸、材质说明
4. **应用场景是展示不是规范** — 画出 IP 在名片上的效果，但没有告诉用户怎么用
5. **缺少错误用法** — 没有任何"禁止这样做"的内容

这些是 Brand Brain 的 Manual Quality Score 需要识别的核心风险模式。
