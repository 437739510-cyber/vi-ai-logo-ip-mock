## 一、Layout Engine 定位

### 设计哲学

```
AI（DeepSeek / Manus / GPT 等）
  ↓ 生成内容（文案、描述、结构）
Layout Engine
  ↓ 执行设计（排版、布局、规范）
VI 手册
```

- AI 模型是可替换的——今天用 DeepSeek，明天用 Manus，后天用 GPT
- Layout Engine 是**不可替换的**——它是品牌的设计规则系统
- Layout Engine 不负责"创意"，它负责"把创意变成规范的视觉呈现"

### 与传统方案的区别

| 方案 | 问题 |
|------|------|
| 当前模板拼接 | 所有品牌共用模板，页面僵硬，缺乏差异化 |
| AI 直接生成布局 | 不可控，风格不稳定，品牌一致性差 |
| **Layout Engine** | **规则驱动，品牌自适应，输出稳定可控** |

### 页面分类体系

#### 按功能分 5 类

| 分类 | 说明 | 示例页面 |
|------|------|---------|
| **品牌形象页** | 展示品牌调性，以视觉冲击为主 | 封面、感谢观看 |
| **品牌内容页** | 传递品牌信息，以阅读体验为主 | 品牌理念、品牌故事、品牌口号 |
| **规范展示页** | 展示设计规范，以信息清晰为主 | 色彩规范、字体规范、Logo 制图 |
| **应用展示页** | 展示应用场景，以效果呈现为主 | 办公应用、包装系统、营销展示 |

#### 按布局密度分 3 类

| 密度 | 说明 | 示例 |
|------|------|------|
| **高视觉** | 图片为主，文字为辅 | 封面、品牌形象页 |
| **均衡** | 图文各半，内容与视觉并重 | 品牌故事、应用展示 |
| **高信息** | 文字/规范为主，图片为辅助说明 | 色彩规范、字体规范 |

#### 布局模式决定因素

```
品牌类型 × 行业特征 × 页面分类 = 布局模式
```

例如：
- 科技行业 + 品牌形象页 → 大留白、几何分割、冷色调
- 餐饮行业 + 品牌形象页 → 温暖质感、有机形状、暖色调
- 教育行业 + 规范展示页 → 清晰网格、多色标注、说明性图示

## 三、Logo 规则

### 核心原则

**Logo 永不重绘。** AI 只能缩放、移动、排版围绕。不能改色、加效果、替换字体。

### Logo 放置规则

| 页面类型 | Logo 位置 | 尺寸规则 | 说明 |
|---------|----------|---------|------|
| 封面 | 居中偏下 / 黄金分割点 | 页面宽度的 30%-50% | 封面主视觉 |
| 品牌内容页 | 左上角 / 右上角 | 页面宽度的 8%-15% | 品牌标识 |
| 规范展示页 | 左上角 | 页面宽度的 6%-10% | 角标展示 |
| 应用展示页 | 场景内嵌入 | 符合场景比例 | 实景融入 |
| IP 规范页 | 左上角或隐藏 | 页面宽度的 6%-8% | 避免干扰 IP |

### Logo 安全空间

```
Logo 周围保留至少等于 Logo 高度 30% 的空白区域
禁止任何文字、图形、装饰侵入安全空间
```

### Logo 背景适配

| 背景类型 | Logo 处理 |
|---------|----------|
| 纯白/浅色背景 | 使用标准彩色 Logo |
| 纯黑/深色背景 | 使用反白 Logo |
| 图片背景（高对比区域） | 加半透明遮罩层后放置标准 Logo |
| 图片背景（低对比区域） | 使用反白 Logo + 轻微投影 |

### Logo 在 A4 页面中的比例

```
页面尺寸：595.28 × 841.89 pt (A4)
封面 Logo：建议宽度 180-300 pt
内页 Logo：建议宽度 50-90 pt
```

## 四、IP 规则

### 核心原则

**已有 IP 永不重绘。** AI 只能识别、保护、规范化应用延展。不能改脸型、颜色、材质、比例。

### IP 放置规则

| 页面类型 | IP 位置 | 尺寸规则 |
|---------|--------|---------|
| 封面（含 IP 时） | Logo 旁或下方 | Logo 高度的 60%-80% |
| IP 规范页 — 基础形象 | 页面中央 | 页面高度的 50%-70% |
| IP 规范页 — 表情系统 | 网格排列（3×3） | 每格 25%-30% 网格宽度 |
| IP 规范页 — 场景延展 | 场景背景中 | 符合场景比例 |
| 应用展示页（含 IP 时） | 场景内 | 自然融入 |

### IP 与 Logo 的共存规则

```
同一页面中：
  - Logo 处于主导地位（品牌标识）
  - IP 处于辅助地位（品牌角色）
  - IP 尺寸不超过 Logo 的 80%
  - 间距至少为 Logo 高度的 50%
```

### IP 展示规范

```
必须展示：
  - 正面标准视图
  - 侧面/半侧面视图（至少 1 个角度）
  - 品牌色值标注（RGB / CMYK / HEX）
  - 最小使用尺寸限制

建议展示：
  - 表情系统（至少 4 种基础表情）
  - 场景应用（至少 2 个场景）
  - 角色设定（性格标签 / 背景故事简述）
```

## 五、图片规则

### 图片来源

| 来源 | 用途 | 处理方式 |
|------|------|---------|
| 品牌方上传 | Logo、IP、产品图、门店图 | 原图引用，不修改 |
| 通义万相生成 | 背景图、装饰图 | 按需生成，可替换 |
| 预留接口 | Flux、DALL·E、Midjourney | 统一 Provider 接口 |

### 图片使用规则

| 用途 | 建议尺寸 | 格式 | 说明 |
|------|---------|------|------|
| 背景图（封面） | A4 全幅 (2480×3508px) | PNG/JPG | 分辨率 ≥ 300dpi |
| 背景图（内页） | 页面局部区域 | PNG/JPG | 需预留内容空间 |
| Logo 引用 | 原图缩放 | PNG（推荐透明底） | 不修改像素 |
| IP 引用 | 原图缩放 | PNG（推荐透明底） | 不修改像素 |
| 产品图 | 原图或裁剪 | JPG | 保持比例 |

### 图片区域分配

```
品牌形象页（封面类）：
  图片区域：60%-80% 页面面积
  内容区域：20%-40% 页面面积

均衡页面：
  图片区域：40%-50% 页面面积
  内容区域：50%-60% 页面面积

高信息页面：
  图片区域：20%-35% 页面面积
  内容区域：65%-80% 页面面积
```

### 图片质量要求

```
背景图片：
  - 无文字、无 Logo（由 Layout Engine 叠加）
  - 与品牌色系协调
  - 不包含敏感元素
  - 留出主要内容区域（中心或黄金分割）

Logo / IP 图片：
  - 透明背景优先
  - 清晰度 ≥ 300dpi
  - 颜色模式：RGB（屏幕展示）
```

## 六、留白规则

### 页面边距

| 页面类型 | 上下边距 | 左右边距 | 说明 |
|---------|---------|---------|------|
| 封面 | 10%-15% | 10%-15% | 视觉冲击优先，留白可破 |
| 品牌内容页 | 8%-12% | 10%-15% | 阅读体验优先 |
| 规范展示页 | 6%-10% | 8%-12% | 信息密度较高 |
| 应用展示页 | 8%-12% | 8%-12% | 均衡布局 |
| IP 规范页 | 8%-12% | 10%-15% | IP 展示为主 |

### 元素间距

```
标题与正文间距：页面高度的 3%-5%
段落间距：正文行高的 1.5-2 倍
图文间距：图片宽度的 5%-10%
分组间距：页面宽度的 5%-8%
网格间距：网格单元宽度的 3%-5%
```

### 留白类型

```
主动留白：为突出核心内容保留的空白（封面、品牌形象页）
功能性留白：为可读性保留的空白（规范展示页）
结构化留白：为信息分组保留的空白（应用展示页）
装饰性留白：为视觉节奏保留的空白（品牌故事页）
```

### 留白与品牌调性

```
高端/奢侈品牌 → 留白偏多（40%-60% 空白）
科技/创新品牌 → 留白中等（30%-50% 空白）
大众/亲民品牌 → 留白适度（20%-40% 空白）
文化/艺术品牌 → 留白灵活（根据内容动态调整）
```

## 七、字体规则

### 字体层级

| 层级 | 用途 | 建议字号 (pt) | 字重 | 行高 |
|------|------|--------------|------|------|
| H0 | 封面大标题 | 48-72 | Bold / ExtraBold | 1.1-1.2 |
| H1 | 页面主标题 | 24-36 | Bold | 1.2-1.3 |
| H2 | 章节标题 | 16-22 | SemiBold | 1.3-1.4 |
| H3 | 子标题 / 标注 | 12-16 | Medium | 1.4-1.5 |
| Body | 正文内容 | 9-12 | Regular | 1.5-1.8 |
| Caption | 说明文字 / 脚注 | 7-9 | Regular / Light | 1.4-1.6 |
| Label | 色值 / 参数标注 | 6-8 | Regular | 1.3-1.4 |

### 中文字体方案

| 场景 | 主选字体 | 备选字体 | 说明 |
|------|---------|---------|------|
| 封面标题 | 思源黑体 Heavy / Noto Sans SC Black | 阿里巴巴普惠体 | 视觉冲击力 |
| 内页标题 | 思源黑体 Bold / Noto Sans SC Bold | 等线 | 清晰可读 |
| 正文内容 | 思源宋体 / Source Han Serif SC | 宋体 | 正式感 |
| 标注说明 | 思源黑体 Regular / Noto Sans SC Regular | 微软雅黑 | 信息清晰 |
| 英文/数字 | Helvetica / Inter | Arial | 搭配中文字体 |

### 字体实现方案

```
V1 建议：
  使用可变字体（Variable Fonts），一个字体文件覆盖所有字重
  推荐：Noto Sans SC Variable / 思源黑体 Variable

技术选型选项：
  A. 本地字体文件（打包部署）— 最稳定，但增加构建体积
  B. CDN 引用（Google Fonts / 阿里云 CDN）— 需要网络，但减少体积
  C. 系统字体回退（sans-serif / serif）— 兼容但不可控

V1 推荐：A + C 组合
  - 内置 Noto Sans SC Bold / Regular 两个字重（文件约 2-5MB）
  - 回退到 system-ui / sans-serif
```

### 字体可读性规则

```
正文最小字号：9pt（印刷） / 12px（屏幕）
行高最小比例：1.5 倍字号
每行最大长度：60-75 字符（中文：25-35 字）
段落间距：至少 1 倍行高
强调文本：使用字重变化，避免使用纯色（品牌色）
禁用：全大写中文排版
```

## 八、色彩规则

### 色彩来源

```
色彩数据来自：
  Step 1 品牌分析输出的 colorSystem 对象
  包含：
    - primary（品牌主色，如 #1A73E8）
    - secondary（辅助色，如 #34A853）
    - accent（强调色，如 #FBBC04）
    - 可选的 extended palette

Layout Engine 的任务：
  将品牌色应用于页面设计，而非决定品牌色
```

### 色彩分配规则

| 用途 | 来源 | 占比 | 说明 |
|------|------|------|------|
| 页面背景 | 白色 / 浅灰 / 品牌色 10% 透明度 | 70%-80% | 保持品牌感但不喧宾夺主 |
| 主标题 | 品牌主色（primary） | 5%-10% | 标题强调 |
| 正文 | 深灰（#333-666） | 10%-15% | 可读性优先 |
| 强调元素 | 品牌强调色（accent） | 3%-5% | 点缀/引导 |
| 分割线/边框 | 品牌主色 20% 透明度 | 1%-2% | 结构分割 |
| 标注/脚注 | 浅灰（#999-AAA） | <1% | 次要信息 |

### 品牌色应用规则

```
品牌色使用限制：
  - 主色用于：标题、关键信息、品牌元素、导航元素
  - 辅助色用于：次级标题、图表、色块区域
  - 强调色用于：按钮、链接、重要标注（使用不超过页面的 5%）
  - 禁用：大面积的纯品牌色背景（除非封面设计需要）

品牌色渐变：
  - 允许主色 → 浅色渐变作为页面装饰
  - 避免高饱和度渐变组合（如红→绿）
  - 推荐单色渐变（主色 → 主色 + 30% 白）
```

### 色彩对比度规则

```
可读性最低要求（WCAG AA）：
  - 正文与背景对比度 ≥ 4.5:1
  - 大标题与背景对比度 ≥ 3:1
  - 标注信息与背景对比度 ≥ 3:1

品牌色在白色背景上：
  - 饱和度过高（饱和度 > 80%）时，降低至 70% 后再用于正文
  - 明度过低（明度 < 30%）时，增加 20% 明度后再用于背景
```

### 色彩标注格式

```
每本手册的色彩页必须包含：
  - RGB 值（屏幕使用）
  - HEX 值（网页使用）
  - CMYK 值（印刷使用）
  - 色块展示（至少 30×30pt）
  - 使用说明（主色用于 / 辅色用于）
```

## 九、自动布局规则

### 网格系统

```
基准网格：
  - 列数：12 列网格
  - 列间距：16pt
  - 行基线：8pt 网格
  - 页面边距：24-48pt（根据页面类型）

页面划分区域：
  ┌─────────────────────────────────┐
  │  顶部区域（标题 / 品牌标识）       │  - 页眉区：8%-15% 页面高度
  ├─────────────────────────────────┤
  │                                 │
  │  内容区域                        │  - 主内容区：60%-75% 页面高度
  │  (图文 / 规范 / 展示)            │
  │                                 │
  ├─────────────────────────────────┤
  │  底部区域（页码 / 脚注）           │  - 页脚区：5%-8% 页面高度
  └─────────────────────────────────┘
```

```
变体：
  D1 — 大图 + 侧边说明（产品展示）
  D2 — 2 图并排 + 底部说明（对比展示）
  D3 — 全幅场景图 + 叠加标注（场景沉浸）
  D4 — 多图网格（3×2/3×3）（系列展示）
```

#### 模式 E：IP 规范页布局

```
┌──────────────────────┐
│ 标题 ──────────────    │
├──────────────────────┤
│                        │
│     IP 基础形象         │
│     （正面，中央大图）    │
│                        │
│  角色名称 / 性格标签     │
├──────────────────────┤
│ 页脚 / 页码            │
└──────────────────────┘
变体：
  E1 — 大 IP 图 + 角色信息（基础形象页）
  E2 — 表情网格（IP 表情系统展示）
  E3 — IP 场景 + 尺寸标注（场景延展）
  E4 — IP 三视图（正面/侧面/背面）
```

### 布局选择规则

```
输入参数：
  - pageType: 页面类型（cover / content / spec / application / ip）
  - brandType: 品牌类型（科技/餐饮/教育/时尚/等）
  - contentDensity: 内容密度（高视觉/均衡/高信息）
  - hasImage: 是否有可用图片
  - imageCount: 图片数量

选择逻辑：
  1. 按 pageType 筛选候选模式
  2. 按 brandType 调整模式参数（色彩/留白/间距）
  3. 按 contentDensity 选择具体变体
  4. 按 hasImage/imageCount 降级或调整

降级规则：
  - 无图片 → 使用纯文字布局 + 装饰元素
  - 单张图片 → 使用左/右图文布局
  - 多张图片 → 使用网格布局
  - 无 IP → 跳过 IP 布局变体
```

### 页面比例

```
所有页面统一为 A4 竖版比例：1:√2（宽:高 = 1:1.414）
渲染尺寸：2480 × 3508 px (300dpi)
```

## 十、页面评分规则

### 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 品牌一致性 | 25% | 是否符合品牌调性、色彩正确 |
| 排版质量 | 20% | 留白、对齐、层级、间距 |
| 可读性 | 20% | 字体大小、对比度、行距 |
| 视觉平衡 | 15% | 图文比例、视觉重心、对称/不对称 |
| 规范正确性 | 10% | 色值标注是否正确、尺寸标注是否规范 |
| 资产保护 | 10% | Logo/IP 是否被正确引用（未重绘） |

### 评分细则

#### 品牌一致性（25 分）

```
- 品牌色使用正确（5 分）
- 品牌调性在视觉中体现（5 分）
- Logo 放置符合规范（5 分）
- 字体选择统一（5 分）
- 行业特征在设计中体现（5 分）
```

#### 排版质量（20 分）

```
- 页面边距符合规范（5 分）
- 元素间距均匀（5 分）
- 对齐方式一致（5 分）
- 视觉层级清晰（5 分）
```

#### 可读性（20 分）

```
- 字号层级清晰（5 分）
- 对比度达标（5 分）
- 行距合适（5 分）
- 每行长度适中（5 分）
```

#### 视觉平衡（15 分）

```
- 图文比例协调（5 分）
- 视觉重心稳定（5 分）
- 页面上下左右均衡感（5 分）
```

#### 规范正确性（10 分）

```
- 色值标注格式正确（3 分）
- 尺寸标注规范（3 分）
- 字体名称/规格标注正确（4 分）
```

#### 资产保护（10 分）

```
- Logo 未被 AI 重绘（5 分）
- IP 未被 AI 重绘（5 分）
```

### 评分等级

| 分数 | 等级 | 处理方式 |
|------|------|---------|
| 90-100 | Excellent | 直接输出 |
| 75-89 | Good | 输出 + 记录改进点 |
| 60-74 | Acceptable | 输出 + 标记可优化项 |
| 40-59 | Needs Revision | 不输出，标记需修改 |
| < 40 | Failed | 不输出，标记为生成失败 |

## 十一、Layout Engine 架构

### 数据流

```
输入:
  {
    pageType: "cover",
    brandData: {
      colors: { primary, secondary, accent },
      logo: { url, width, height },
      ip: { url, width, height } | null,
      fonts: { family, weights },
      industry: "technology",
      brandTone: "professional"
    },
    content: {
      title: "品牌名称",
      subtitle: "...",
      body: "...",
      images: [ ... ]
    },
    layoutMode: "A1" | "auto"  // auto = 由 Engine 选择
  }

Layout Engine 处理:
  1. 布局选择器（根据 brandData + pageType 选择模式）
  2. 内容适配器（将 content 映射到布局插槽）
  3. 样式解析器（将 brandData 转换为 CSS/SVG 属性）
  4. 渲染器（生成最终 SVG / HTML / PDF）

输出:
  {
    svg: "...",      // 可直接渲染的 SVG
    metadata: {
      layoutMode: "A1",
      score: 85,
      warnings: []
    }
  }
```

### 模块划分

```
Layout Engine
├── 布局选择器 (LayoutSelector)
│   ├── 模式库 (PatternLibrary)
│   ├── 品牌适配器 (BrandAdapter)
│   └── 降级处理器 (FallbackHandler)
├── 内容适配器 (ContentAdapter)
│   ├── 文本处理器 (TextProcessor)
│   ├── 图片处理器 (ImageProcessor)
│   └── 资产保护器 (AssetGuard)
├── 样式解析器 (StyleResolver)
│   ├── 色彩解析器 (ColorResolver)
│   ├── 字体解析器 (FontResolver)
│   └── 间距计算器 (SpacingCalculator)
├── 渲染器 (Renderer)
│   ├── SVG 生成器 (SvgGenerator)
│   ├── HTML 生成器 (HtmlGenerator) [Future]
│   └── PDF 合成器 (PdfComposer)
└── 评分器 (Scorer)
    ├── 品牌一致性检查 (BrandConsistencyCheck)
    ├── 排版质量检查 (TypographyCheck)
    └── 资产保护检查 (AssetProtectionCheck)
```

### 技术选型建议

| 组件 | V1 建议 | 理由 |
|------|---------|------|
| 布局描述 | JSON 配置 | 与模型无关，易于扩展 |
| 渲染引擎 | SVG（Sharp 渲染为 PNG） | 兼容现有流程 |
| 字体方案 | 内置 Noto Sans SC (Bold + Regular) | 最稳定 |
| 评分器 | 规则引擎 + 可选 AI 辅助 | 先规则确保可靠 |
