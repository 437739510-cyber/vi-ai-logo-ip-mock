03
整理 PRD

START

复制给 Codex 的命令
深色代码框可整段复制；看不懂文件名，就直接追问 Codex。

一开始最容易卡在“不知道该说什么”。这一页提供可直接复制的第一批指令。

① 环境检查：请先不要写代码。帮我检查当前项目运行环境，包括 Node.js、npm、
Git、package.json 和项目目录结构。请告诉我是否缺少依赖、应该执行哪条命令、
执行后应该看到什么结果。

② 创建规则：请创建 agents.md，写清本项目的协作规则：先输出计划再改文件；每
次大改前提醒我提交 Git；不能把 API Key 写进代码；修改后必须给出 npm run 
build 或本地预览验证方法。

③ 生成PRD：请创建 PRD.md，项目名为“云上样板间”。内容包括：首页、小区筛选、
房源详情、3D看房、地图配套、AI推荐、收藏、咨询留资、Mock数据、技术栈、验收
标准和未来可扩展点。

卡住就问：“我不知道这个文件应该放在哪里，请告诉我在项目根目录怎么创建 agents.md 和 PRD.md，并给我最简单
的检查方式。”

把“云上样板间”拆成页面、功能、数据、验收标准，
让 Codex 有稳定方向。

验收：根目录出现 agents.md 与 PRD.md；后续任务都能引用这两个文件。

PLAN

计划模式不是形式主义。它能提前看见改动范围、验证方式和失败回滚路径。

复制给 Codex 的命令
深色代码框可整段复制；不会判断计划时，直接让 Codex 改成表格。

④ 计划模式：请阅读 PRD.md 和 agents.md，先进入计划模式，不要写代码。输出 
TASK_PLAN.md，列出每一步目标、预计改动文件、验证命令、风险、需要我确认的
问题。等我确认后再开始第一步。

⑤ 拆分任务：请把任务拆成可独立执行的窗口：视觉首页、房源详情、3D看房、地图
API、AI推荐、收藏留资、部署上线。每个窗口写清输入文件、输出文件、验收标准和
不能碰的范围。

⑥ 上下文压缩：请生成 PROJECT_CONTEXT.md，包含项目目标、技术栈、已完成
内容、当前待办、关键文件、环境变量占位名、运行命令、常见错误和下一窗口开工
提示。

卡住就问：“我不知道哪些任务能开新窗口，请按风险从低到高排序，并告诉我课堂上先做哪一个。”

验收：TASK_PLAN.md 可读；任务能拆；PROJECT_CONTEXT.md 能让新窗口接手。
